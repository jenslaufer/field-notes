# Claude Code built me a $273/day directory

> Frey Chu builds boring, profitable **online directories** with Claude Code —
> e.g. a **luxury restroom-trailer directory built in ~4 days for under $250**,
> now **~$273/day**. The model: **$2K–$10K/mo** passive on **$200–$1,000**
> startup cost. The hard part isn't the site (Claude Code does that) — it's
> **sourcing data nobody else has**. Data scarcity = the moat.

## Pick the keyword (this is the whole bet)

- Target **"near me" intent** keywords — high commercial intent, local.
- Sweet spot: **30K–100K monthly searches**, **keyword difficulty < 20**.
- The directory must help users **save time, save money, or make money**.
- Worked examples from the episode: **jetski rentals** (74K/mo, KD 13),
  **dog beaches** (60.5K/mo, KD 4), **pediatric dentists** (201K/mo, KD 1).

## The 7-step data pipeline (the real work)

- **1. Scrape raw data** — Google Maps + niche sites via **Crawl4AI**. Get the
  base list (names, locations).
- **2. Clean** — strip obvious errors, dupes, junk rows.
- **3. Enrich with Claude Code** — the nuanced pass: fill gaps, normalize,
  categorize at scale across thousands of records.
- **4. Verify** — cross-check against thousands of source websites for accuracy.
- **5. Extract niche features** — the *filters* that make it a directory, not a
  list: licensing, guided tours, delivery, off-leash, insurance accepted,
  emergency hours. **Price transparency** where data is scarce = the killer
  feature.
- **6. Source images** — with licensing/legal care.
- **7. Map service areas** — geographic relevance so "near me" actually resolves.

## Build + structure

- **Claude Code builds the site**; pairing it with Crawl4AI saves "thousands of
  hours" of manual data work.
- Spin up **SEO pages per related keyword** (city × service combinations) to fan
  out long-tail traffic.
- The **data moat**: in industries where info is scarce, *being the only source
  of clean, filterable, price-transparent data* is the defensibility.

## Monetization models (stack several)

- **Lead generation** — sell qualified leads to listed businesses (Parting.com,
  APlaceForMom.com).
- **Vertical SaaS** — upsell listed businesses a tool.
- **Ads / affiliate** — AdSense + affiliate links (GasBuddy = millions/yr).
- **Agency, memberships, marketplace, debit-card / data plays** as later layers.

## Marketing tips

- **Keyword-first, product-second.** The SEO opportunity (volume × low KD × "near
  me" intent) *is* the go-to-market — pick the keyword before you build.
- **Programmatic SEO:** generate one page per city/service combo from your data —
  long-tail "near me" traffic compounds for near-zero marginal cost.
- **Lead the niche on price transparency** where competitors hide pricing — it's
  both the user hook and the search-result differentiator.
- **Low CAC by design:** organic search intent means you're harvesting demand,
  not buying it. Passive once indexed.

## Why this matters for us

- **Distribution-first via SEO intent:** validate demand with keyword volume +
  difficulty *before* building — same "test offer×audience first" discipline as
  launch-kit idea validation.
- **Data as the moat** is a reusable wedge: a scraped-+-enriched dataset behind a
  thin filterable UI is a copyable, low-maintenance product recipe.
- **Claude Code + Crawl4AI** is a concrete cheap-build stack we can mirror for
  any data-backed micro-tool or lead funnel.
- Multiple **monetization layers** (lead gen → SaaS → affiliate) show how one
  asset funds several revenue lines.
