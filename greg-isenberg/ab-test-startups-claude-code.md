# A/B-Test Startups in Seconds with Claude Code

> Greg and Amir take a business idea from zero to a validated, live A/B-tested
> landing page in a single Claude Code session. The stack: **Idea Browser MCP**
> (context) → **Paper** (design) → **HumbleLytics** (analytics + experiments).
> The pitch: "99.999% of people have no clue this stack exists" — Amir compares
> it to early Facebook ads when clicks cost five cents. Some agencies charge
> **$5K–$10K/month** to run exactly this.

## The 4-phase workflow

### 1. Strategy & context — Idea Browser MCP (~00:00–06:10)
- **Connect Idea Browser via MCP** so Claude Code reads it straight in the terminal.
- **Pull the business foundation**: ICP, positioning, offer definition, growth strategy.
- **Persist over time** — the context tracks how the idea evolves, so each
  iteration makes better decisions instead of starting from a blank prompt.

### 2. Visual design — Paper (~06:10–20:06)
- **Paper replaces the Figma-to-dev handoff** — design visually, stay synced to code.
- **Generate multiple component variants**, then pick the strongest.
- **Concrete edits made live**: refine the hero headline, swap a pricing section
  for an **ROI calculator**, build a reusable component library.

### 3. Deploy & instrument — HumbleLytics (~20:06–28:38)
- **Deploy the page** and drop in **HumbleLytics** with no front-end coding.
- **Track conversions and clicks** in real time from one dashboard.

### 4. Live experiments — Claude Code hits the API (~28:38–32:44)
- **Claude Code calls the HumbleLytics API** to spin up an A/B test — no redeploy.
- **Dynamic headline swap** per traffic segment; watch performance live.
- Example test headline: *"Every lost deal started with an objection your rep
  wasn't ready for."*

## The example build
- **Niche**: B2B sales enablement.
- **Core offer / lead magnet**: an objection-handling framework.
- **Funnel goal**: capture leads, optimized via dynamic headline testing.

## Marketing tips
- **Test the headline first.** It's the highest-leverage variable; HumbleLytics
  swaps it per segment with zero redeploy, so you learn in hours not weeks.
- **Replace pricing with an ROI calculator** when the buyer's pain is quantifiable
  — interaction beats a static price table for B2B lead capture.
- **Lead with the prospect's loss, not your feature** — the winning variant named
  the buyer's failure ("a lost deal," "an objection your rep wasn't ready for").
- **Arbitrage the unknown stack.** Almost nobody runs MCP-context → Paper →
  HumbleLytics end-to-end. Same dynamic Amir frames as "Facebook ads at 5¢/click."
- **Productize it.** The exact workflow is sold as managed service for $5K–$10K/mo —
  the build is cheap, the packaging is the margin.

## Why this matters for us
- **Launch-kit blueprint**: idea context → landing page → live experiment is the
  exact funnel we want. The MCP-context layer (persistent ICP/offer) is the part
  most teams skip and the part that makes iterations compound.
- **No-redeploy A/B** via an analytics API means we can ship one page and test
  copy continuously — a copyable conversion recipe.
- **Paper-style sync** (design ↔ code) avoids the Figma handoff tax; worth
  evaluating for our own landing pages.
